## IPOTESI

- Confidenza alta: `LastTaskResult: 0` is the scheduler-level closure signal for this step.
- Confidenza alta: `phase7_executive_status.ps1` can legitimately observe `task_state=Running` when it is executed inside the scheduled runner before the parent task has fully exited.
- Confidenza alta: when `task_state=Running`, `task_last_result` is not the final result of the current run; it is the previous Scheduler result or an in-progress code.

## DECISIONE

- Mark daily scheduler automation as `CLOSED / ACTIVE`.
- Keep retention active in the scheduled production runner.
- Keep `-RetentionWhatIf` only for manual validation.
- Add explicit `task_result_interpretation` to the executive status export:
  - `IN_PROGRESS_PREVIOUS_RESULT` when the task is still running.
  - `IDLE_FINAL_RESULT` when the task is idle and Scheduler result is final.
- Do not treat `task_state=Running` inside the executive status file as a failure by itself.

## ROI

- Closes the recorder-independent evidence continuity loop.
- Reduces operational ambiguity after recorder incidents.
- Prevents future false NO-GO interpretation caused by reading Scheduler state while the task is still executing.
- Avoids new BMS logic until observability is trustworthy.
